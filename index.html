About SecureStego Vault

How It Works

1

Upload & Encrypt

Select your file and enter a 6-digit PIN. The file is encrypted using AES-256-GCM encryption.

2

Key Embedding

Your encryption key is hidden inside a procedurally generated abstract image using steganography.

3

Share Securely

Share the encrypted file and key image. Send the PIN through a separate, secure channel.

Security Features

  • AES-256-GCM: Military-grade encryption with authenticated encryption
  • PBKDF2-SHA256: 100,000 iterations for strong key derivation
  • Zero-Knowledge: All processing happens in your browser - nothing sent to servers
  • LSB Steganography: Encryption keys hidden in images using least significant bit encoding
  • HMAC Integrity: SHA-256 HMAC ensures data hasn't been tampered with
  • Open Source: Fully transparent code - audit it yourself on GitHub

Best Practices

Important Security Tips:

  • Never share your PIN through the same channel as your files (e.g., don't email all three together)
  • Use a strong, random 6-digit PIN - avoid birthdays or simple patterns
  • Anyone with all three components (encrypted file, key image, and PIN) can decrypt your file
  • Delete the original unencrypted file and encryption materials after successful transfer
  • For maximum security, share the PIN verbally or through a separate encrypted messenger

Frequently Asked Questions

Is my data sent to any server?

No. All encryption, decryption, and image generation happen entirely in your browser using the Web Crypto API. Your files never leave your device.

What if I forget my PIN?

Unfortunately, if you lose your PIN, the encrypted file cannot be recovered. The encryption is designed to be unbreakable without the correct PIN.

What file types are supported?

All file types up to 1GB are supported - documents, images, videos, archives, and more.

Can I use this on mobile devices?

Yes! SecureStego Vault works on all modern mobile browsers (iOS Safari 14+, Android Chrome 90+).

Is the key image visible in the generated art?

No. The encryption metadata is hidden using LSB steganography, making it invisible to the human eye and undetectable without specialized analysis.
